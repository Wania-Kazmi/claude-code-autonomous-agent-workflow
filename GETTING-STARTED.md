## ðŸš€ First-Time Usage

### Step 1: Create Your Requirements File

```bash
# Copy the example
cp requirements/example.md requirements/my-app.md

# Edit it with your project requirements
# (Use your preferred editor)
```

### Step 2: Run Autonomous Build

```bash
claude "/sp.autonomous requirements/my-app.md"
```

This will:
1. Verify Spec-Kit-Plus installation (must be pre-installed)
2. Analyze your requirements
3. Generate missing skills/agents/hooks (if needed)
4. Create project specification
5. Build implementation plan
6. Generate code with tests
7. Run quality reviews

**Note**: `/sp.autonomous` assumes Spec-Kit-Plus is already installed. The `.claude/` and `.specify/` directories must exist before running this command.

### Step 3: Resume Work After Days/Weeks

When you start a **new conversation** later:

```bash
# See your saved TODOs from last session
bash .claude/scripts/resume-work.sh

# Ask Claude: "Please restore these TODOs"
```

## âš ï¸ Important Notes

### Clean Slate vs Session Recovery

- **First time using this boilerplate?** You start with a clean slate (no TODOs)
- **Returning after days?** Run `bash .claude/scripts/resume-work.sh`
- **Working with teammates?** Commit `.specify/todos.json` to share TODO state

### What's Ignored by Git

These files are session-specific and NOT committed:
- `.specify/todos.json` (your active TODO state)
- `.specify/todo-history/` (historical snapshots)
- `.specify/workflow-state.json` (workflow phase tracking)
- `.claude/logs/*.log` (usage logs)

See `.gitignore` for full list.

### What's Committed

These files ARE shared via git:
- All skills (`.claude/skills/`)
- All agents (`.claude/agents/`)
- All hooks (`.claude/hooks.json`)
- Documentation
- Settings and rules
- Example files (`.specify/todos.example.json`)

## ðŸ› Troubleshooting

### "No saved TODOs found"
**Normal for first-time use!** TODOs are created when you first use the TodoWrite tool.

### "Unknown skill: X"
Some skills are generated on-demand during the workflow. This is expected.

### Build Fails
1. Try: `claude "/build-fix"`
2. Check: `claude "/code-review"`
3. Review errors in `.claude/logs/`

### TODOs Not Merging
Check that `Stop` hook is configured in `.claude/settings.json` (it is by default).
