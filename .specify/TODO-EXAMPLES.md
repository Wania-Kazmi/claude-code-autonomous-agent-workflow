## üí° Tips for Writing Good TODOs

### ‚úÖ DO:
- Be specific: "Implement user authentication with JWT" vs "Do auth"
- Include acceptance criteria when relevant: "Add test coverage (80%+ target)"
- Use action verbs: "Implement", "Create", "Set up", "Add", "Write"
- Break large tasks into smaller ones
- Keep activeForm in present continuous tense: "Implementing..." not "Implement..."

### ‚ùå DON'T:
- Be vague: "Fix stuff", "Make it work"
- Create tasks too large: "Build entire app"
- Use unclear terminology: "Do the thing"
- Forget to update status as work progresses

### üìä Status Guidelines:
- **completed**: Task is 100% done, tested, and verified
- **in_progress**: Currently working on this task
- **pending**: Not started yet, waiting in queue
